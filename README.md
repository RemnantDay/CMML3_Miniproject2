# Benchmarking Doublet-Detection Approaches in Simulated Single-Cell RNA Sequencing Data with Varying Doublet Rates and Sequencing Depths

## Introduction of the repository
The original code for generating single-cell RNA-sequencing data using scDesign3, benchmark, and data visualization, as well as all relevant data, are available here. 

## Overview
The presence of doublets is a key confounder in the analysis of single-cell RNA sequencing data. Here, we benchmarked five existing doublet-detection methods (scDblFinder, cxds, bcds, Hybrid and DoubletFinder) on simulated scRNA-seq data with varying doublet rates and sequencing depths. We discovered that all methods except cxds achieved high accuracy and would benefit more from increasing doublet rates than from increasing sequencing depths. scDblFinder outperformed in discovering heterotypic doublets with the highest computational efficiency.

## Method summary
Simulated scRNA-seq datasets were generated using scDesign3 v1.7.6 with the default reference dataset (Song et al., 2024). Four annotated immune cell types were retained, and 2,000 balanced initial singlets were simulated using the top 500 highly variable genes to reduce memory use. Two benchmark settings were designed. For the doublet-rate benchmark, sequencing depth was fixed at 2,000 UMI per singlet, and doublet rates were set to 5%, 10%, 20%, 30% and 40%. For the sequencing-depth benchmark, doublet rate was fixed at 20%, and singlet library sizes were set to 500, 1,000, 2,000, 4,000 and 8,000 UMI. Artificial doublets were generated by randomly pairing singlets and summing their UMI profiles, followed by removal of the parent singlets. Five doublet-detection methods, scDblFinder, cxds, bcds, Hybrid and DoubletFinder, were benchmarked using recommended settings(McGinnis et al., 2019; Bais and Kostka, 2020; Germain et al., 2022). Performance was assessed by AUPRC, AUROC, running time and manually calculated homotypic/heterotypic recall. Figures were generated in R using ggplot2, dplyr and tidyr.

### 1. Generate_data_figure1.R

The R file used to generate 5 .rds files with different doublet rates (5%, 10%, 20%, 30%, 40%), related to Figure 1.

Five .rds files generated by the code are located in the **simulated_doublet_datasets_Figure1/** folder. *Figure1_dataset_summary.csv* file summarizes the five datasets.

### 2. Generate_data_figure2.R

The R file used to generate 5 .rds files with different sequencing depths (0.5k, 1k, 2k, 4k, 8k), related to Figure 2.

Five .rds files generated by the code are located in the **simulated_doublet_datasets_Figure2/** folder. *Figure2_depth_dataset_summary.csv* file summarizes the five datasets.

### 3. Benchmark_data_1.R

The R file used to benchmark 5 datasets with different doublet rates, related to Figure 1. We used five doublet detection methods, including scDblFinder, cxds, bcds, hybrid, and DoubletFinder.

Six .csv files generated by the code are located in the **simulated_doublet_datasets_Figure1/benchmark_Figure1** folder. "predictions_DoubletFinder.csv", "predictions_scDblFinder_random.csv", and "predictions_scds_cxds_bcds_hybrid.csv" files are the direct output of five doublet detection methods. "metrics_DoubletFinder.csv", "metrics_scDblFinder_random.csv", and "metrics_scds_cxds_bcds_hybrid.csv" files contain the processed information evaluating the performance.

### 4. Benchmark_data_2.R

The R file used to benchmark 5 datasets with different sequencing depth, related to Figure 2. We used five doublet detection methods, including scDblFinder, cxds, bcds, hybrid, and DoubletFinder.

Six .csv files generated by the code are located in the **simulated_doublet_datasets_Figure2/benchmark_Figure2_depth** folder. "predictions_DoubletFinder.csv", "predictions_scDblFinder_random.csv", and "predictions_scds_cxds_bcds_hybrid.csv" files are the direct output of five doublet detection methods. "metrics_DoubletFinder.csv", "metrics_scDblFinder_random.csv", and "metrics_scds_cxds_bcds_hybrid.csv" files contain the processed information evaluating the performance.

### 5. Visualization_figure1.R

The R file used to visualize the data after running the "Benchmark_data_1.R" file, related to Figure 1b-g. The figures are available in the main text.

### 6. Visualization_figure2.R

The R file used to visualize the data after running the "Benchmark_data_2.R" file, related to Figure 2a-f. The figures are available in the main text.
